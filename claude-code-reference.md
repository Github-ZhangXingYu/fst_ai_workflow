# Claude Code Official Configuration Reference

> Source: code.claude.com/docs | Generated from official documentation

---

## 1. .claude Directory Structure

Claude Code reads configuration from the `.claude/` directory in your project and `~/.claude/` in your home directory.

### Project-level (`<project>/.claude/`)

```
<project>/
├── CLAUDE.md                         # Project instructions (also works at .claude/CLAUDE.md)
├── .mcp.json                         # Project-scoped MCP servers
├── .worktreeinclude                  # Gitignored files to copy into new worktrees
└── .claude/
    ├── settings.json                 # Permissions, hooks, and configuration
    ├── settings.local.json           # Personal overrides (gitignored)
    ├── CLAUDE.md                     # Alternative location for project instructions
    ├── rules/                        # Topic-scoped instructions
    │   ├── testing.md
    │   └── api-design.md
    ├── skills/                       # Reusable prompts (directory + SKILL.md)
    │   └── <skill-name>/
    │       ├── SKILL.md              # Entrypoint (required)
    │       ├── reference.md          # Supporting files (optional)
    │       └── scripts/
    ├── commands/                     # Single-file prompts invoked with /name
    │   └── <command-name>.md
    ├── output-styles/                # Custom system-prompt sections
    ├── agents/                       # Specialized subagents
    │   └── <agent-name>.md
    ├── workflows/                    # Dynamic workflow scripts (.js)
    ├── agent-memory/                 # Subagent persistent memory (auto-generated)
    │   └── <agent-name>/
    │       └── MEMORY.md
    └── hooks/                        # Hook scripts (optional convention)
```

### Global-level (`~/.claude/`)

```
~/
├── .claude.json                      # App state, UI preferences, personal MCP servers
└── .claude/
    ├── CLAUDE.md                     # Personal preferences across every project
    ├── settings.json                 # Default settings for all projects
    ├── keybindings.json              # Custom keyboard shortcuts
    ├── themes/                       # Custom color themes (.json)
    ├── projects/                     # Auto memory (auto-generated)
    │   └── <project>/memory/
    │       ├── MEMORY.md
    │       └── <topic-files>.md
    ├── rules/                        # User-level rules for every project
    ├── skills/                       # Personal skills available in every project
    ├── commands/                     # Personal commands available in every project
    ├── output-styles/                # Personal output styles
    ├── agents/                       # Personal subagents available in every project
    ├── workflows/                    # Personal dynamic workflows
    └── agent-memory/                 # Persistent memory for subagents with memory:user
```

### File badge meanings

| Badge | Meaning |
|-------|---------|
| `committed` | Should be committed to version control |
| `gitignored` | Automatically gitignored when Claude Code creates it |
| `local` | Never committed, applies to your machine only |
| `Claude writes` | Auto-generated by Claude, safe to edit or delete |

## 2. settings.json

### Configuration Scopes (precedence order)

| Scope | Location | Priority | Shared? |
|-------|----------|----------|---------|
| **Managed** | Server-managed / MDM / `managed-settings.json` | 1 (highest) | Yes (IT-deployed) |
| **CLI flags** | `--model`, `--permission-mode`, etc. | 2 | No |
| **Local** | `.claude/settings.local.json` | 3 | No (gitignored) |
| **Project** | `.claude/settings.json` | 4 | Yes (committed) |
| **User** | `~/.claude/settings.json` | 5 (lowest) | No |

**Scalar settings** (`model`, `outputStyle`, etc.): use the most specific value.
**Array settings** (`permissions.allow`, `hooks`, etc.): merge across all scopes.

### Example settings.json

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write"
          }
        ]
      }
    ]
  },
  "env": {
    "CLAUDE_CODE_ENABLE_TELEMETRY": "1"
  },
  "model": "claude-sonnet-4-6",
  "outputStyle": "default"
}
```

### Key Settings Reference

| Key | Description | Example |
|-----|-------------|---------|
| `model` | Default model for this project | `"claude-sonnet-4-6"` |
| `fallbackModel` | Fallback when primary is unavailable | `"claude-haiku-4-5"` |
| `outputStyle` | System-prompt style | `"default"` |
| `defaultShell` | Default shell for Bash tool | `"bash"` |
| `permissions.allow` | Tools/commands allowed without prompting | `["Bash(git status)", "Bash(npm test *)"]` |
| `permissions.ask` | Tools/commands requiring confirmation | `["Bash(git push *)"]` |
| `permissions.deny` | Tools/commands always denied | `["Bash(rm -rf *)", "WebFetch"]` |
| `permissions.defaultMode` | Default permission mode | `"default"` |
| `permissions.additionalDirectories` | Extra directories for acceptEdits mode | `["../docs/"]` |
| `hooks` | Lifecycle hooks configuration | See Hooks section |
| `env` | Environment variables for every session | `{"MY_KEY": "value"}` |
| `autoCompactEnabled` | Auto-compact when context is full | `true` |
| `autoMemoryEnabled` | Enable auto memory | `true` |
| `autoMemoryDirectory` | Custom auto memory location | `"~/my-memory"` |
| `respectGitignore` | Respect .gitignore for file operations | `true` |
| `cleanupPeriodDays` | Session cleanup period | `30` |
| `editorMode` | Editor mode | `"normal"` |
| `statusLine` | Custom status line configuration | See docs |
| `skillOverrides` | Control skill visibility | `{"deploy": "off"}` |
| `spinnerTipsEnabled` | Show spinner tips | `true` |
| `disableAllHooks` | Disable all hooks | `false` |
| `disableBundledSkills` | Disable bundled skills | `false` |

### Permission Modes

| Mode | Behavior |
|------|----------|
| `default` | Standard permission checking with prompts |
| `acceptEdits` | Auto-accept file edits and common filesystem commands |
| `auto` | Background classifier reviews commands |
| `dontAsk` | Auto-deny permission prompts |
| `bypassPermissions` | Skip permission prompts (use with caution) |
| `plan` | Plan mode (read-only exploration) |
| `manual` | Alias for `default` (v2.1.200+) |

### When edits take effect

Most settings are hot-reloaded when files change. A few are read once at session start:
- `model` (use `/model` to switch mid-session)
- `outputStyle` (rebuilds on `/clear` or restart)

## 3. Hooks

### Hook Events

| Event | When it fires | Can block? |
|-------|---------------|------------|
| `SessionStart` | Session begins or resumes | No |
| `Setup` | `--init-only` or `--init` / `--maintenance` in `-p` mode | No |
| `UserPromptSubmit` | User submits a prompt | Yes |
| `UserPromptExpansion` | User-typed command expands into a prompt | Yes |
| `PreToolUse` | Before a tool call executes | Yes |
| `PermissionRequest` | Permission dialog appears | Yes |
| `PermissionDenied` | Tool call denied by auto mode classifier | No (can return `{retry: true}`) |
| `PostToolUse` | After a tool call succeeds | No |
| `PostToolUseFailure` | After a tool call fails | No |
| `PostToolBatch` | After a full batch of parallel tool calls | Yes |
| `Notification` | Claude Code sends a notification | No |
| `MessageDisplay` | While assistant message text is displayed | No |
| `SubagentStart` | Subagent is spawned | No |
| `SubagentStop` | Subagent finishes | Yes |
| `TaskCreated` | Task is being created via `TaskCreate` | Yes |
| `TaskCompleted` | Task is being marked completed | Yes |
| `Stop` | Claude finishes responding | Yes |
| `StopFailure` | Turn ends due to API error | No (output ignored) |
| `TeammateIdle` | Agent team teammate about to go idle | Yes |
| `InstructionsLoaded` | CLAUDE.md or `.claude/rules/*.md` loaded | No |
| `ConfigChange` | Configuration file changes during session | Yes |
| `CwdChanged` | Working directory changes | No |
| `FileChanged` | Watched file changes on disk | No |
| `WorktreeCreate` | Worktree being created | Yes |
| `WorktreeRemove` | Worktree being removed | No |
| `PreCompact` | Before context compaction | Yes |
| `PostCompact` | After context compaction completes | No |
| `Elicitation` | MCP server requests user input during tool call | No |
| `ElicitationResult` | User responds to MCP elicitation | No |
| `SessionEnd` | Session terminates | No |

### Matcher Patterns

| Matcher value | Evaluated as | Example |
|---------------|-------------|---------|
| `"*"`, `""`, or omitted | Match all | fires on every occurrence |
| Only letters, digits, `_`, `-`, spaces, `,`, and `\|` | Exact string(s) | `Bash`, `Edit\|Write`, `Edit, Write` |
| Contains any other character | JavaScript RegExp (unanchored) | `^Notebook`, `mcp__memory__.*` |

**Event-specific matcher fields:**

| Event | Matcher filters | Example values |
|-------|----------------|----------------|
| `PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `PermissionRequest`, `PermissionDenied` | tool name | `Bash`, `Edit\|Write`, `mcp__.*` |
| `SessionStart` | how session started | `startup`, `resume`, `clear`, `compact` |
| `SessionEnd` | why session ended | `clear`, `resume`, `logout`, `prompt_input_exit`, `other` |
| `SubagentStart`, `SubagentStop` | agent type | `general-purpose`, `Explore`, `Plan`, custom names |
| `PreCompact`, `PostCompact` | what triggered compaction | `manual`, `auto` |
| `ConfigChange` | configuration source | `user_settings`, `project_settings`, `local_settings`, `policy_settings` |
| `FileChanged` | literal filenames to watch | `.envrc\|.env` |
| `Notification` | notification type | `permission_prompt`, `idle_prompt`, `auth_success`, etc. |
| `StopFailure` | error type | `rate_limit`, `overloaded`, `authentication_failed`, etc. |

**No matcher support (always fires):** `UserPromptSubmit`, `PostToolBatch`, `Stop`, `TeammateIdle`, `TaskCreated`, `TaskCompleted`, `WorktreeCreate`, `WorktreeRemove`, `MessageDisplay`, `CwdChanged`

### MCP Tool Matching

MCP tools follow the pattern `mcp__<server>__<tool>`:
- `mcp__memory__create_entities`
- `mcp__github__search_repositories`
- `mcp__memory__.*` matches all tools from the `memory` server
- `mcp__.*__write.*` matches any write tool from any server

Plugin-bundled MCP tools: `mcp__plugin_<plugin-name>_<server-name>__<tool>`

### Hook Handler Types

#### type: "command"

```json
{
  "type": "command",
  "command": "bash .claude/hooks/security_check.sh",
  "args": [],
  "timeout": 600,
  "async": false,
  "asyncRewake": false,
  "shell": "bash",
  "if": "Bash(git push *)",
  "statusMessage": "Checking security...",
  "once": false
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `command` | Yes | Shell command to execute |
| `args` | No | Argument list (exec form, no shell) |
| `timeout` | No | Seconds before canceling (default: 600) |
| `async` | No | Run in background without blocking |
| `asyncRewake` | No | Run in background, wake Claude on exit 2 |
| `shell` | No | `"bash"` or `"powershell"` |

**Exit codes:**
- `0`: No decision, normal permission flow continues
- `2`: Block the operation
- Other: Non-blocking warning

#### type: "http"

```json
{
  "type": "http",
  "url": "https://hooks.example.com/claude",
  "timeout": 600
}
```

#### type: "mcp_tool"

```json
{
  "type": "mcp_tool",
  "server": "my-mcp-server",
  "tool": "my_tool",
  "timeout": 600
}
```

#### type: "prompt"

```json
{
  "type": "prompt",
  "prompt": "Check if this edit introduces security vulnerabilities...",
  "timeout": 30
}
```

Returns `{"ok": true/false, "reason": "..."}`

#### type: "agent"

```json
{
  "type": "agent",
  "prompt": "Verify all unit tests pass. Check: $ARGUMENTS",
  "timeout": 60
}
```

### Hook Input/Output Format

**Example stdin input (PreToolUse):**

```json
{
  "session_id": "uuid-string",
  "cwd": "/path/to/project",
  "hook_event_name": "PreToolUse",
  "tool_name": "Bash",
  "tool_input": {
    "command": "git push origin main"
  },
  "transcript_path": "/tmp/transcript.jsonl"
}
```

**Block operation (exit code 2):**

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "Destructive command blocked by hook"
  }
}
```

**Modify tool input:**

```json
{
  "hookSpecificOutput": {
    "modifyInput": {
      "command": "git push origin main --dry-run"
    }
  }
}
```

**Add context (Stop/SubagentStop):**

```json
{
  "hookSpecificOutput": {
    "additionalContext": "Remember to run tests before committing."
  }
}
```

**Return `{retry: true}` (PermissionDenied):**

```json
{
  "hookSpecificOutput": {
    "retry": true
  }
}
```

### Hook Configuration Example

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "if": "Bash(rm *)",
            "command": "${CLAUDE_PROJECT_DIR}/.claude/hooks/block-rm.sh"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write"
          }
        ]
      }
    ],
    "Stop": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/stop-verify.sh"
          }
        ]
      }
    ],
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/prompt-validate.sh",
            "timeout": 30
          }
        ]
      }
    ],
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/session-init.sh"
          }
        ]
      }
    ]
  }
}
```

## 4. Skills

Skills extend Claude's capabilities. Each skill is a directory with `SKILL.md` as the entrypoint.

### Directory Layout

```
.claude/skills/<skill-name>/
├── SKILL.md              # Main instructions (required)
├── reference.md          # Detailed reference (optional)
├── examples.md           # Usage examples (optional)
└── scripts/
    └── helper.sh         # Bundled scripts (optional)
```

### SKILL.md Format

```yaml
---
name: my-skill                        # Display name (optional, directory name is default)
description: What this skill does and when to use it.  # Recommended
when_to_use: Additional trigger context
argument-hint: <issue-number>         # Shown in autocomplete
arguments: [issue, branch]            # Named positional arguments
disable-model-invocation: true        # Only manual /skill-name invocation
user-invocable: false                 # Hide from / menu (Claude-only)
allowed-tools: Read Grep Bash         # Pre-approved tools
disallowed-tools: AskUserQuestion     # Removed tool access
model: sonnet                         # Override model
effort: high                          # Override effort level
context: fork                         # Run in isolated subagent
agent: Explore                        # Agent type for fork context
hooks:                                # Skill-scoped hooks
  PreToolUse:
    - matcher: "Write"
      hooks:
        - type: command
          command: "bash validate.sh"
paths:                                # Auto-load when files match
  - "src/api/**/*.ts"
  - "**/*.test.ts"
shell: bash                           # Shell for !`command` syntax
---

# Skill instructions (markdown)

Your instructions here...
```

### Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | No | Display label, 64 chars max, lowercase + hyphens |
| `description` | Recommended | What the skill does; Claude uses this to auto-invoke. Max 1536 chars combined with `when_to_use` |
| `when_to_use` | No | Additional trigger phrases or examples |
| `argument-hint` | No | Autocomplete hint, e.g. `[issue-number]` |
| `arguments` | No | Named positional arguments for `$name` substitution |
| `disable-model-invocation` | No | `true` = only manual invocation; also prevents preloading into subagents |
| `user-invocable` | No | `false` = hide from `/` menu |
| `allowed-tools` | No | Tools Claude can use without prompting (space/comma separated or YAML list) |
| `disallowed-tools` | No | Tools removed while skill is active |
| `model` | No | Override model (accepts aliases: `sonnet`, `opus`, `haiku`, `fable`, or full model ID, or `inherit`) |
| `effort` | No | Override effort: `low`, `medium`, `high`, `xhigh`, `max` |
| `context` | No | `fork` to run in isolated subagent |
| `agent` | No | Agent type for `context: fork` (`Explore`, `Plan`, `general-purpose`, or custom) |
| `hooks` | No | Skill-scoped lifecycle hooks |
| `paths` | No | Glob patterns for auto-activation |
| `shell` | No | `bash` (default) or `powershell` for `!` commands |

### String Substitutions

| Variable | Description |
|----------|-------------|
| `$ARGUMENTS` | All arguments passed |
| `$ARGUMENTS[N]` | Specific argument by 0-based index |
| `$N` | Shorthand for `$ARGUMENTS[N]` (`$0`, `$1`, ...) |
| `$name` | Named argument from `arguments` frontmatter |
| `${CLAUDE_SESSION_ID}` | Current session ID |
| `${CLAUDE_EFFORT}` | Current effort level |
| `${CLAUDE_SKILL_DIR}` | Directory containing SKILL.md |
| `${CLAUDE_PROJECT_DIR}` | Project root directory |

### Dynamic Context Injection

The `` !`command` `` syntax runs shell commands before the skill is sent to Claude:

```yaml
---
name: pr-summary
description: Summarize changes in a pull request
---

## Pull request context
- PR diff: !`gh pr diff`
- PR comments: !`gh pr view --comments`
- Changed files: !`gh pr diff --name-only`
```

Multi-line commands use fenced blocks with ` ```! `:

````markdown
```!
node --version
npm --version
git status --short
```
````

### Invocation Control

| Frontmatter | You can invoke | Claude can invoke | When loaded |
|-------------|----------------|-------------------|-------------|
| (default) | Yes | Yes | Description always in context, full skill when invoked |
| `disable-model-invocation: true` | Yes | No | Description not in context, full skill only on manual invoke |
| `user-invocable: false` | No | Yes | Description always in context, full skill when invoked |

### Legacy Commands (still supported)

A file at `.claude/commands/<name>.md` creates `/<name>` the same way. Same frontmatter support. If a skill and command share a name, the skill takes precedence.

## 5. Subagents (`.claude/agents/`)

### File Format

```markdown
---
name: code-reviewer
description: Reviews code for quality and best practices
tools: Read, Glob, Grep
disallowedTools: Write, Edit
model: sonnet
permissionMode: default
maxTurns: 50
skills:
  - api-conventions
  - error-handling-patterns
mcpServers:
  - playwright:
      type: stdio
      command: npx
      args: ["-y", "@playwright/mcp@latest"]
  - github
hooks:
  PostToolUse:
    - matcher: "Write|Edit"
      hooks:
        - type: command
          command: "${CLAUDE_PROJECT_DIR}/.claude/hooks/validators/check.py"
  Stop:
    - matcher: "*"
      hooks:
        - type: agent
          prompt: |
            Review the implementation for correctness.
          tools: Read, Bash
memory: project
background: true
effort: high
isolation: worktree
color: purple
initialPrompt: Check the codebase for any obvious issues
---

You are a code reviewer. When invoked, analyze the code and provide
specific, actionable feedback on quality, security, and best practices.
```

### Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Unique identifier (lowercase letters and hyphens) |
| `description` | Yes | When Claude should delegate to this subagent |
| `tools` | No | Allowed tools (inherits all if omitted) |
| `disallowedTools` | No | Tools to remove from inherited/specified list |
| `model` | No | `sonnet`, `opus`, `haiku`, `fable`, full model ID, or `inherit` (default) |
| `permissionMode` | No | `default`, `acceptEdits`, `auto`, `dontAsk`, `bypassPermissions`, `plan`, `manual` |
| `maxTurns` | No | Maximum agentic turns before stopping |
| `skills` | No | Skills to preload into subagent context at startup |
| `mcpServers` | No | MCP servers available (inline definition or string reference) |
| `hooks` | No | Lifecycle hooks scoped to this subagent |
| `memory` | No | Persistent memory scope: `user`, `project`, or `local` |
| `background` | No | `true` to always run as background task |
| `effort` | No | Effort level override |
| `isolation` | No | `worktree` for isolated git worktree |
| `color` | No | Display color: `red`, `blue`, `green`, `yellow`, `purple`, `orange`, `pink`, `cyan` |
| `initialPrompt` | No | Auto-submitted first turn when running as main session agent |

### Scope Priority

| Location | Scope | Priority |
|----------|-------|----------|
| Managed settings | Organization-wide | 1 (highest) |
| `--agents` CLI flag | Current session | 2 |
| `.claude/agents/` | Current project | 3 |
| `~/.claude/agents/` | All projects | 4 |
| Plugin `agents/` directory | Where plugin enabled | 5 (lowest) |

### Built-in Subagents

| Agent | Model | Tools | Purpose |
|-------|-------|-------|---------|
| `Explore` | Inherits (capped at Opus) | Read-only | Codebase exploration |
| `Plan` | Inherits | Read-only | Plan mode research |
| `general-purpose` | Inherits | All | Complex multi-step tasks |

## 6. CLAUDE.md

### Locations (load order, broadest to most specific)

| Scope | Location |
|-------|----------|
| Managed policy | macOS: `/Library/Application Support/ClaudeCode/CLAUDE.md`; Linux: `/etc/claude-code/CLAUDE.md`; Windows: `C:\Program Files\ClaudeCode\CLAUDE.md` |
| User | `~/.claude/CLAUDE.md` |
| Project | `./CLAUDE.md` or `./.claude/CLAUDE.md` |
| Local | `./CLAUDE.local.md` |

All discovered files are concatenated into context (not overriding). Content is ordered from filesystem root down to working directory. Within each directory, `CLAUDE.local.md` is appended after `CLAUDE.md`.

### Rules (`.claude/rules/`)

Markdown files in `.claude/rules/` discovered recursively. Rules without `paths` frontmatter load at launch. Rules with `paths` frontmatter load when matching files enter context.

```markdown
---
paths:
  - "src/api/**/*.ts"
---

# API Development Rules

- All endpoints must include input validation
- Use the standard error response format
```

### Import Additional Files

CLAUDE.md can import other files using `@path/to/file` syntax:

```markdown
See @README for project overview.
- git workflow @docs/git-instructions.md
```

Imports skip code spans and fenced blocks. Use backticks to reference without importing: `` `@README` ``.

## 7. MCP Configuration (`.mcp.json`)

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
```

Personal MCP servers go in `~/.claude.json` (use `claude mcp add --scope user`).

## 8. .mcp.json Schema

MCP servers can be defined with:
- `type`: `stdio` (default), `http`, `sse`, `ws`
- `command` + `args`: for `stdio` transport
- `url`: for `http`, `sse`, `ws` transports
- `env`: environment variables

### Subagent MCP Server Inline Definition

Subagents can define MCP servers inline via `mcpServers` frontmatter:

```yaml
---
mcpServers:
  - playwright:
      type: stdio
      command: npx
      args: ["-y", "@playwright/mcp@latest"]
  - github  # References already-configured server
---
```

## 9. Global Config (`~/.claude.json`)

Stores app state that doesn't belong in settings.json:

```json
{
  "autoConnectIde": true,
  "externalEditorContext": true,
  "mcpServers": {
    "my-tools": {
      "command": "npx",
      "args": ["-y", "@example/mcp-server"]
    }
  }
}
```

Contains: OAuth session, theme, per-project trust decisions, personal MCP servers, UI toggles.
